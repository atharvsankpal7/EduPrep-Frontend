## 2025-02-12 - Prevent High-Frequency Timer Component Re-renders
**Learning:** In the test engine architecture, components that update at high frequencies (like `SectionTimer` ticking every second) cause heavy UI re-renders up the component tree if not properly isolated. Even if a child component updates its own state, if its parent (`TestEngineShell`) re-renders for other reasons and passes down inline functions (e.g., `onExpire={() => void handleTimerExpired().catch(handleAutoSubmitFailure)}`), it breaks referential equality and forces the high-frequency component to re-render unnecessarily, compounding performance issues.
**Action:** When implementing or modifying high-frequency updating components, always wrap them in `React.memo` and ensure the parent component passes down referentially stable callbacks (using `useCallback`) to preserve the memoization optimization.
