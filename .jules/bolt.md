## 2024-10-30 - High-Frequency State Updates and Callback Stability
**Learning:** In highly complex components that manage test engine state (like `TestEngineShell`), passing an inline callback (e.g., `onExpire={() => {...}}`) to a timer component that ticks every second causes the timer to re-render constantly. `SectionTimer` itself manages high-frequency state, but if it receives a new reference for `onExpire` on every re-render of its parent, it cannot benefit from `React.memo()`. The combination of parent-level state updates and missing memoization on the child with inline functions led to unnecessary render cycles across the timer.
**Action:** For high-frequency updating components (like timers, scroll handlers, etc.), always wrap the component in `React.memo` AND ensure all callbacks passed to it are referentially stable using `useCallback`.
