## 2026-03-02 - Memoization in High-Frequency Update Components
**Learning:** In applications with high-frequency updating components (like `SectionTimer` ticking every second), passing inline arrow functions from parent components (e.g., `<SectionTimer onExpire={() => { ... }} />`) causes the child to re-render on every tick, even if wrapped in `React.memo`. This is because the inline function's reference changes on every parent render, breaking memoization.
**Action:** Always extract inline functions passed to memoized components into referentially stable callbacks using `useCallback`, ensuring that both `React.memo` and the stable callback work together to prevent unnecessary renders.
## 2026-03-02 - Memoization in High-Frequency Update Components (Timer State)
**Learning:** In applications with high-frequency updating components (like `Timer` ticking every second), maintaining the state (e.g., `timeLeft`) in the parent component causes the entire parent component (and all its children) to re-render on every tick. Additionally, passing an unmemoized callback (e.g., `onTimeUp={() => { ... }}`) to the child component causes the child's `useEffect` to tear down and recreate the `setInterval` on every tick, destroying performance.
**Action:** Always extract high-frequency local state down into the child component itself. If the parent needs to respond to events (like `onTimeUp`), pass a referentially stable callback using `useCallback`, ensuring that `React.memo` and the stable callback work together to prevent unnecessary renders and `useEffect` teardowns.
